1. POST /api/farms – Create a Farm

Description:
Creates a new farm with basic information and an uploaded banner image.

Request Type: multipart/form-data

Body fields:

FarmName (string) – required

FarmDesc (string)

Phone (string)

Area (string)

FarmerId (uuid) – required

Province (string)

District (string)

Ward (string)

Detail (string)

FarmBanner (file, binary) – required (farm banner image)

Response (200):

success (boolean)

message (string)

data containing:

farmId (uuid)

farmName (string)

bannerUrl (string)

phone (string)

area (string)

2. GET /api/farms/me – Get Farm Owned by Current User

Description:
Returns the farm associated with the authenticated user.

Request Type:

No parameters

No request body

Response (200):

Farm data belonging to the logged-in farmer.

3. POST /api/farms/{farmId}/certificate – Upload a Farm Certificate

Description:
Uploads a single certificate file for a specific farm.

Path parameter:

farmId (uuid) – required

Request Type: multipart/form-data

Body fields:

FarmId (uuid) – must match the path

Certificate (file, binary) – required

Response (200):

Upload successful confirmation

4. DELETE /api/farms/{farmId}/certificate – Delete Farm Certificate

Description:
Deletes the certificate associated with the given farm.

Path parameter:

farmId (uuid) – required

Response (200):

Certificate deletion success

5. PUT /api/farms/{farmId}/certificates – Update/Replace Farm Certificate

Description:
Replaces the certificate of the given farm with a new file.

Path parameter:

farmId (uuid) – required

Request Type: multipart/form-data

Body fields:

FarmId (uuid)

Certificate (file, binary) – new certificate file

Response (200):

Certificate update success

1. GET /api/categories

Retrieve all available product categories.

Request

Method: GET

URL: /api/categories

Parameters: None

Successful Response (200 OK)

Returns a list of categories with their details.

{
  "success": true,
  "message": "Success",
  "data": [
    {
      "id": "uuid",
      "categoryName": "string",
      "categoryDesc": "string",
      "illustrativeImageUrl": "string (URL)",
      "isDelete": true
    }
  ]
}

2. GET /api/products

Retrieve products with optional filtering by keyword, category, or location.

Request

Method: GET

URL: /api/products

Query Parameters (Optional):

Parameter	Type	Description
searchTerm	string	Keyword to search by product name
categoryId	UUID	Filter products by category
location	string	Filter products by farmer or product region
Example Request
GET /api/products?categoryId=525f4919-a03f-4ce7-3301-08de2965ff1e

Successful Response (200 OK)

Returns a list of products matching the filters.

{
  "success": true,
  "message": "Retrieving data successfully!",
  "data": [
    {
      "id": "uuid",
      "productName": "string",
      "productAttribute": "string",
      "productDesc": "string",
      "categoryId": "uuid",
      "createdAt": "2025-11-22T01:41:54.7469695"
    }
  ]
}
